Unique features


I really wanted this project to be a showcase of my developmental prowess and ability to create not only beautiful UI, great design but also to bring a concept that was industry standard and viable. I am a big believer in test driven development and believe it makes for a far superior product.

Unit testing

As the project is built on the Angular Framework testing was particularly easy to implement, I have a Jasmine test suite running through Karma at the moment.

At the time of writing I have 1100 lines of application code and 1600 lines of unit testing for the front end.

As testing is centric to my development process, i have an istanbul code coverage tool running on my front end to test my unit test coverage, I take a lot of pride in keeping this as green as feasibly possible (100% is best!)



REST endpoint testing

To ensure that my backend node server is running across all the API's that I require I have built a custom test runner from the vows and API-easy test suites.



Above is an example of one of the back end tests. What makes this so cool is the custom create and destory DB commands. When setup_db is ran, the db is cleaned and you can specify in JSON/Object literal format the setup data to load into the database (or reference a JSON file) which makes testing endpoints based on different database states a breeze!



Here is just a little of the command line output that shows which endpoint is being tested, and the number of tests. As you can see, I don't like to have any holes in my testing, so I have a TODO function in the code that causes a little output prompt reminder that there are unwritten tests, a nice little reminder!
