Design

The design steps

My design process incorporated a lot of the steps of a full UI / UX process, this was of course not as in depth as I would have liked, but gives an idea of how I like to design products

  1. Data review: What data do we have? Ratings? Images? What types of content are we delivering? How much data is there?
  2. Existing design review: Look at other designs / UI mock ups that people have made for design inspiration. I identified what was liked / disliked about each design, and ideas that I wanted to incorporate into my own design
  3. Sketching: Whilst looking at other ideas I would sketch different ideas that I had, so I wouldn't forget them, but also so that I could see on paper whether I liked the direction the design it was heading.

Technology choice

Angular

I chose to build the app with Angular as this is a framework I've been working with a considerable amount in the past few years, so building flexible and clean architectures is a very quick process for me currently.

SASS

Sass is used for the styling, this helps to keep things neat within the component-based approach, all colours / shadows / transitions are controlled by single global variables meaning that the style guide of the architecture is flexible. Update one value for the colour and this will propogate to the rest of the app with a single change [6]

Build / Features

Performance

The search to the foursquare API [3] is debounced so that the user is given 500ms to stop typing before a request is made, giving the user time to enter their search term fully before getting their resultset.

Animations

On load the left panel animates in from the left

Component based approach

The interface is broken up into separate components, for instance the results is comprised of restaurant tiles, with this approach the content can be moved around VERY easily, keeping the architecture flexible and agile friendly [6]

Known defects

Drag / Click on map

Drag / Clicking is not currently working, as the z-index of the map container is affected in order to overlay the search, an alternative was not found within the deadline time

Improvements / Out of scope

Infinite scroll

The foursquare app has a "see more results" button at the bottom of their list, this is nice as it segments and paginates which helps the interface load fast, but the user has to click to see the results, an infinite scroll here would be much nicer and not require the user to have to click to see more results, simply scroll.

Typeahead

An autocomplete / typeahead would improve the UX when searching for items

Groupings / Filters

Groupings based on categories and simple filters under the search box

Alternative card based UI

Rather than map centric UI, a card based UI would allow the user to expand the search list to view extra details in a non location centric interface

Responsive UI

The left search panel is currently fixed width meaning smaller resolution devices do not get any meaningful UX
